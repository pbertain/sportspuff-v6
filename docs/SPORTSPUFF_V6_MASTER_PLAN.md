# Sportspuff v6 Master Plan

## Purpose

This document is the working architecture and product plan for Sportspuff v6 after the 12-league scores surface expansion. It consolidates the current Flask/PostgreSQL app, static asset conventions, live `sportspuff-api` integration, visual theme direction, and the proposed achievement/easter egg framework.

Source planning notes folded into this plan:

- `TENNIS_BRANDING_AND_EASTER_EGGS.md`
- `DESIGN_SYSTEM.md`
- `GRAPHICS_BACKLOG.md`

## Current Architecture

Sportspuff v6 is a Flask application backed by PostgreSQL. The app owns team, league, stadium, color, logo filename, standings-adjacent local data, and rendered HTML pages. Live scores, schedules, season info, and most standings are consumed from the external `sportspuff-api` service and proxied through Flask where needed.

Primary application layers:

- Backend app: `app.py`
- Templates: `templates/`
- Shared styling: `static/css/main.css`
- Frontend behavior: mostly inline JavaScript in `templates/index.html`, plus `static/js/contrast.js`
- Local static images: `static/images/`
- Source/logo asset folders: `logos/`, `graphics/`, `stadiums/`
- Database schema/import scripts: `database_schema*.sql`, `import_data*.py`, migration helpers
- Deployment: `.github/workflows/test-and-deploy.yml`, `ansible/`, `deploy.sh`

Key routes and responsibilities:

- `/`: homepage with the scores/schedules surface and league cards.
- `/league/<league_name>`: league detail page, teams grouped by conference/division, with selected live records.
- `/teams`, `/stadiums`, `/team/...`, `/stadium/...`: local data browsing.
- `/api/v1/teams`, `/api/v1/stadiums`: versioned local JSON APIs with envelope, filters, limit, and offset.
- `/curl/v1/teams`, `/curl/v1/stadiums`: terminal-friendly text variants.
- `/api/proxy/all-scores/<date>`: aggregate schedule and score proxy for all 12 leagues.
- `/api/proxy/schedule/<league>/<date>` and `/api/proxy/scores/<league>/<date>`: single-league live data proxies.
- `/api/season-info/<league>`: normalized season/tour phase data, using local constants for some leagues and `sportspuff-api` for others.
- Supplemental proxy endpoints: NFL/MLS/WNBA/MLB records, cricket standings, NHL playoff series, API status.

The homepage schedule surface currently supports these 12 league keys:

`ATP`, `CYCLING`, `IPL`, `MLB`, `MLC`, `MLS`, `NBA`, `NFL`, `NHL`, `WC`, `WNBA`, `WTA`.

The homepage uses a canonical render order matching the league multi-select. When multiple leagues are active it fetches `/api/proxy/all-scores/<date>` and filters client-side. When one league is active it uses the single-league schedule and score proxy endpoints.

## Data Model

The core database model is normalized around:

- `teams`
- `stadiums`
- `leagues`
- `conferences`
- `divisions`

Current local data strengths:

- Team/stadium browsing is mature.
- Team colors and logo filenames are available for major team leagues.
- League/team pages can compute or display standings-like records for MLB, MLS, WNBA, and other supported leagues.

Current limitations:

- The checked-in base schema is older than the live application expectations. App code references additional columns such as `logo_filename`, `league_name_proper`, `team_color_*`, `team_abbreviation`, and champion metadata.
- ATP, WTA, WC, and CYCLING are schedule-surface leagues but are not yet full local league-page data models.
- Event-specific entities such as tennis tournaments, cycling races/stages, and World Cup rounds are currently represented by API payload fields, not local tables.

Planning implication: future schema work should happen through migrations and should distinguish team-based leagues from event/tour-based leagues.

## Static Assets Strategy

Sportspuff currently uses three asset tiers:

- Local runtime assets under `static/images/`, including copied logos, stadium images, and `winner-star.png`.
- Source/raw working assets under `logos/`, `stadiums/`, and `graphics/`.
- Remote production logo URLs at `https://www.splitsp.lat/logos/...`, generated by template filters and frontend constants.

Logo URL conventions in use:

- Team logos: `https://www.splitsp.lat/logos/<league>/<logo_filename>`
- League logos: `https://www.splitsp.lat/logos/<league>/<league>_logo.png`
- Current guessed additions:
  - `https://www.splitsp.lat/logos/wc/wc-logo.png`
  - `https://www.splitsp.lat/logos/atp/atp-logo.png`
  - `https://www.splitsp.lat/logos/wta/wta-logo.png`
  - `https://www.splitsp.lat/logos/cycling/cycling-logo.png`

Recommended canonical asset structure:

```text
static/images/
  logos/
    <league>/
  events/
    tennis/
      balls/
      tournaments/
      mascots/
      badges/
    cycling/
      races/
      jerseys/
      badges/
    wc/
      tournaments/
      badges/
  achievements/
  stadiums/
```

Recommended source asset structure:

```text
graphics/
  grand-slam-assets/
  event-assets/
    tennis/
    cycling/
    wc/
  prompts/
```

Asset naming rules:

- Use lowercase kebab-case filenames.
- Prefer transparent PNG for bitmap UI assets and SVG for badges/icons when licensing and rendering needs allow it.
- Keep official-inspired assets visually distinct from official marks unless license clearance exists.
- Put inspectable production assets under `static/images/...`; keep prompts, references, and experiments under `graphics/...`.

Tennis asset roadmap:

- Official-inspired ball set: ATP navy, WTA purple, Wimbledon green, French Open clay orange, US Open navy, Australian Open blue.
- Wordmarks or event marks where legally safe: French Open, US Open, Australian Open.
- Mascot set for hidden and playful surfaces: Wimbledon, Roland Garros, US Open, Australian Open, ATP, WTA.
- Badges: Career Grand Slam, Tennis Completionist, Puhfph Open, Rain Delay, GOAT Detector.

## OpenAPI Integration Strategy

Sportspuff v6 should treat `sportspuff-api` as the source of truth for live sports data and the Flask app as a UI-oriented cache/proxy layer.

Current integration pattern:

- Base URL comes from `SPORTSPUFF_API_BASE_URL`, defaulting to `https://api.sportspuff.net`.
- Proxy routes avoid browser CORS issues.
- Schedule cache TTL is 300 seconds.
- Score cache TTL is 61 seconds.
- A background refresh thread warms all-score data every 60 seconds for PT, ET, CT, and MT.
- Expired cache can be served while async refresh runs.
- Empty fallback objects are returned for missing schedule/score data to keep the UI renderable.

Current upstream endpoint families:

- `/api/v1/schedule/<league>/<date>?tz=<tz>`
- `/api/v1/scores/<league>/<date>?tz=<tz>`
- `/api/v1/standings/<league>`
- `/api/v1/season-info/<league>`
- `/api/v1/status`

Recommended OpenAPI contract work:

- Keep an OpenAPI spec in the API repo as the formal contract, then mirror the relevant consumer expectations in this repo.
- Define shared game fields across team sports:
  - `game_id`, `league`, `sport`, `game_time`, `game_status`, `is_final`
  - `home_team`, `visitor_team`, `home_score`, `visitor_score`
  - `home_wins`, `home_losses`, `visitor_wins`, `visitor_losses`
- Define optional sport-specific extensions:
  - Basketball/hockey playoff series: `is_playoff`, `series_round`, `series_summary`, `series_total`, `home_series_wins`, `visitor_series_wins`
  - Tennis: `tennis_tournament`, `tennis_round`, optional set-score fields if upstream adds them later
  - Cycling: `cycling_race`, `cycling_stage_label`, `cycling_country`, `cycling_event_label`
  - World Cup: `wc_round_label`, group/knockout metadata
  - Cricket: formatted innings score fields and result strings
- Version the frontend handling by field presence, not only by date or league.
- Add contract fixtures for representative API responses:
  - NBA Finals playoff game
  - ATP/WTA match with tournament and round
  - Cycling stage/race entry
  - WC group and knockout match
  - Offseason league with `last_champion`
  - Tour-active empty day

OpenAPI governance:

- New fields should be additive.
- Removed or renamed fields need a compatibility period.
- Schedule and score responses should agree on `game_id`.
- League keys should stay lowercase in API payloads and be normalized to uppercase only in display code.
- Event/tour sports should not be forced into home/visitor semantics when the sport does not naturally have them.

## Theme System Design

Current state:

- Global CSS custom properties live in `static/css/main.css`.
- The app uses a dark purple base palette, Bootstrap 5, Font Awesome, and many league-specific classes.
- League accent colors are defined as CSS variables for MLB, NHL, NBA, NFL, WNBA, IPL, MLC, and MLS.
- Homepage empty-state banners define additional inline style classes for WC, ATP, WTA, and CYCLING.
- Some critical score UI styling is currently embedded in `templates/index.html`.

Target theme system:

- Move league and event tokens into a single token layer.
- Keep normal UI professional and sports-stat friendly.
- Reserve humor, animations, mascots, and unlocks for secondary or opt-in surfaces.
- Preserve WCAG AA contrast for text and controls.
- Ensure theme changes never hide schedules, teams, scores, records, or navigation.

Recommended token groups:

```css
:root {
  --sp-bg;
  --sp-surface;
  --sp-surface-strong;
  --sp-text;
  --sp-text-muted;
  --sp-border;
  --sp-accent;
  --sp-accent-contrast;
}

[data-league-theme="nba"] { ... }
[data-league-theme="atp"] { ... }
[data-event-theme="wimbledon"] { ... }
[data-event-theme="career-grand-slam"] { ... }
```

Theme scopes:

- Site theme: base Sportspuff identity.
- League theme: NBA, NFL, MLB, NHL, MLS, WNBA, IPL, MLC, WC, ATP, WTA, CYCLING.
- Event theme: Wimbledon, Roland Garros, US Open, Australian Open, Tour de France, World Cup 2026.
- Achievement theme: Career Grand Slam gold, Puhfph Open rainbow/colorblind theme.

Tennis theme token candidates from the design notes:

- `--tennis-atp-navy: #0b214a`
- `--tennis-wta-purple: #6f35ff`
- `--tennis-wimbledon-green: #1f7a3a`
- `--tennis-french-clay: #d96b2b`
- `--tennis-us-open-navy: #002f6c`
- `--tennis-australian-open-blue: #00a3e0`
- `--tennis-career-gold: #d4af37`

Implementation direction:

- Extract homepage score styles from inline `<style>` into `static/css/main.css` or a dedicated scores stylesheet.
- Replace repeated hard-coded league colors with token references.
- Introduce `data-league` and `data-event` attributes on score sections and league pages.
- Build a small client-side theme controller that stores user opt-in themes in `localStorage`.
- Keep server-rendered default styling usable with JavaScript disabled.

## Achievement And Easter Egg Framework

The achievement system should be a progressive enhancement, not a dependency for normal sports browsing.

Core principles:

- No account requirement for phase 1; store local unlocks in `localStorage`.
- Do not interrupt schedule reading or navigation.
- Make all animations dismissible or brief.
- Respect reduced-motion preferences.
- Use original Sportspuff/Puhfph assets for hidden rewards.
- Keep official-inspired tournament assets in normal UI; reserve mascots and playful content for discoveries, badges, 404 pages, and transitions.

Recommended client-side state model:

```json
{
  "version": 1,
  "unlocks": {
    "career-grand-slam": {
      "unlocked_at": "2026-06-06T00:00:00Z"
    }
  },
  "progress": {
    "tennis_ball_sequence": ["australian-open", "roland-garros"]
  },
  "settings": {
    "theme": "sportspuff",
    "motion": "system"
  }
}
```

Initial achievements:

- Career Grand Slam Mode: click tournament balls in Australian Open, Roland Garros, Wimbledon, US Open order. Unlocks gold accents.
- Tennis Completionist: collect ATP, WTA, and all four Grand Slam stamps.
- Puhfph Open: reveal a hidden fifth tournament after completing the tennis set.
- Rain Delay Mode: rare Wimbledon effect with a short rain animation.
- Clay Dust Cursor: subtle Roland Garros drag trail.
- GOAT Detector: hidden footer interaction that opens collectible cards without declaring a winner.
- Tennis Ball Physics: repeated clicks make tournament balls squish, rotate, then fly.

Framework pieces:

- `AchievementStore`: localStorage read/write, migration, reset.
- `AchievementRegistry`: static metadata for achievements, badge assets, and dependencies.
- `EventTracker`: records user interactions such as tournament-ball clicks.
- `UnlockPresenter`: toast/modal/badge animation, with reduced-motion handling.
- `ThemeController`: applies unlockable themes using data attributes.

Future server-side upgrade path:

- Add authenticated profiles only if the product needs cross-device persistence.
- Add a `user_achievements` table later; do not block phase 1 on it.
- Consider anonymous signed export/import for local progress before adding full auth.

## New Sports Product Plan

Phase 1: 12-league homepage surface

- Multi-select shows all 12 leagues alphabetically.
- ATP/WTA render as matchup rows with `vs`, not fake `0-0`.
- Cycling renders as a full-row race/stage banner.
- WC renders standard score rows with round labels.
- Empty banners are league/tour aware.
- Offseason banners prefer `last_champion` when supplied.

Phase 2: League pages for new sports

- Add pages for WC, ATP, WTA, and CYCLING.
- Do not force all four into the current team/division layout.
- WC page should support tournament overview, groups, knockout bracket, match list, and host/stadium links.
- ATP/WTA pages should support active tournament, daily matches, draw/round labels, and Grand Slam branding.
- CYCLING page should support active race, stage list, standings/classifications if the API supplies them, and country/race branding.

Phase 3: Asset hardening

- Confirm final logo paths on `splitsp.lat`.
- Promote approved graphics from `graphics/` into `static/images/events/...`.
- Add fallbacks for missing remote assets.
- Add asset inventory documentation and ownership/licensing notes.

Phase 4: Active-event banner

- Add event-specific banners using event logos or tournament balls:
  - Wimbledon / Roland Garros / US Open / Australian Open
  - Tour de France
  - World Cup 2026
- Use active season/tour phase from `/api/season-info/<league>`.
- Prefer event-specific imagery over generic league logos.

Phase 5: Themes and achievements

- Add event theme tokens.
- Add tennis ball theme switcher.
- Add local achievement framework.
- Add mascot/badge art and hidden unlocks.
